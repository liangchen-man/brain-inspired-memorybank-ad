| **DMAD** (Memory Bank of Anomaly and Normality) | 双记忆库 / 半监督 | 同时维护正常 memory 和异常 memory，利用少量异常样本构建异常记忆库 | 从单正常记忆库扩展为正常+异常双记忆库 | 可视角为"人工反馈后的异常案例库"——Case 2 中人类介入被确认为正常后，应纳入正常库，而非默认全部新样本都是异常 | **不实现**，列为未来工作 |

## 2. 为什么本项目不实现这些 SOTA 方法

### 2.1 课程目标决定了范围边界

本项目的课程目标是：

> 理解工业异常检测中 memory bank 范式的原理，并迁移到真实工位场景，分析静态正常性记忆的局限和动态记忆的必要性。

而不是"在 MVTec AD 上追平全量 SOTA 方法"。第一周四类算法（PaDiM / SPADE / PatchCore / AnomalyDINO）已覆盖 memory bank 的四种核心形式（统计式、样本式、代表性 patch、强特征 token），足以支撑"正常性记忆"概念的建立。

### 2.2 Case 4 已证明瓶颈不是模型大小

Case 4 三方法收敛失效实验（v1 PatchCore+ResNet18、v2 PatchCore+DINOv2、v3 像素可见性基线）共同指向一个结论：**瓶颈不在模型容量或算法先进性，而在采集信号的可观测性**。

- 侧帽缺失的像素信号（~10px 变化）被光照闪烁（σ=5-26）、相机微振（1-3px）、玻璃反射淹没
- DINOv2（518×518, 37×37 feature grid）仍未解决 ~50px 微小缺陷的检测——不是"模型不够强"，而是"特征网格太粗"
- 在这种条件下，换成 DMAD 或 Dinomaly 也不会根本改善——输入信号的 SNR 低于任何模型的检测下限

因此，继续堆前沿方法对于本项目而言是**错配资源**——问题不在模型侧，而在采集侧。

### 2.3 Case 2 已证明问题不是单帧模型强弱

Case 2 的 8 状态 temporal baseline 揭示了另一个维度的限制：

- 同一像素在不同时间对应不同物体（滤芯移动）
- "空位"有双重语义（正常空 vs 异常缺失）
- 人类介入（t=391-395s）在单帧上是极度异常的外观，但在流程中是正常补料
- 滤芯数 4→3→4→3 可逆——单帧检测器无法判断"3 支"是正常还是异常

这些是**单帧 vs. 时序**的根本问题——换更强的单帧模型（如 Dinomaly）同样无法区分"正常的空"和"异常的空"，因为它缺少时间上下文。

### 2.4 训练成本和算力受限

| 方法 | 需要的资源 | 本项目的约束 |
|------|-----------|------------|
| DMAD | 需要异常样本标注、双记忆库维护 | Case 4/Case 2 仅有视频、无异常 GT 标注 |
| SimpleNet | 需要训练去噪网络、调噪声参数 | 不适合以审计为核心的视频分析 |
| Dinomaly | 需要大量多类数据训练统一模型 | MVTec AD 仅两类可用，视频无 GT |
| 扩散模型 | 需要大量正常样本训练、推理延迟高 | RTX 3050 4GB 不适合，工业实时性不满足 |

### 2.5 已有实验已满足报告需求

当前项目的第一周+第二周实验已经形成完整的论证链：

```
MVTec AD 静态 memory bank (AUROC 0.98+) 
    → Case 4: 采集可观测性限制（三方法收敛失败）
    → Case 2: 单帧不够，需要时序状态（8-state baseline + HUMAN_INTERACTION）
    → 类脑计算解释：长期记忆 + 短期记忆 + 人工反馈门控
```

向这个论证链中添加 DMAD/Dinomaly 实验并不能使其更强——因为瓶颈已在模型能力之外被充分诊断。

## 3. 前沿方法对本项目的启发（作为"未来工作"引用）

### 3.1 DMAD → 异常案例库的建立

DMAD 的双记忆库设计提示：在真实工业部署中，人工确认过的异常案例应被保留为"异常记忆库"（而非当作正常样本吞入正常库）。对应本项目 Case 2 的人工反馈机制——当系统检测到 `HUMAN_INTERACTION` 状态时，不是直接报警，而是请求人工确认"这是正常补料吗？"，确认后纳入正常模式而非当作异常。

### 3.2 SimpleNet → 正常记忆库的边界

SimpleNet 的合成异常思想提示：仅靠正常样本的特征聚类（MVTec AD coreset），其边界取决于 coreset 的代表性。如果正常状态分布很宽（如 Case 2 包含空位、运动模糊、驻留），coreset 特征覆盖的密度就不均匀——可能需要显式标注"什么不是正常"来收紧边界。

### 3.3 Dinomaly → 从逐类到统一的方向

Dinomaly 的"统一模型"方向与本项目 AnomalyDINO 实验（DINOv2 pretrained token memory）属于同一技术路线——更强的预训练特征可以减少对每类独立 memory bank 的依赖。项目的 AnomalyDINO pretrained vs non-pretrained 消融（0.99559 vs 0.33427）已从消融角度证实了这一方向的合理性。

### 3.4 AnomalyGPT → 语义解释的未来

本项目 Case 2 中对 qwen3-vl-plus 的使用（"Please describe what is different between these two frames"）本质上是手动版的 AnomalyGPT——用 VLM 替代人工审计。未来如果部署自动化的 VLM 审计流水线，可以替代本项目中人工逐帧确认的步骤。
