### 0.3 事件时序重构（候选）

```

    ↓
t≈360s    亮度过渡区开始（相机自动曝光调整？）
    ↓
t≈380s    第4支滤芯被自动装置或人类移除（具体时间未采样到）
    ↓  3支滤芯停留在导轨上

t=397s    人类离开，4支滤芯全在位
    ↓
t≈403-405s  第4支滤芯再次被移除（自动或人手，采样间隔内发生）
    ↓  3支滤芯保持到视频结束

```

### 0.4 关键洞察

1. **人工介入持续时间：约 4 秒**（t=389-397s 整个窗口，实际手部接触约 t=391-395s）
2. **2秒采样间隔仅捕获 2-3 帧**——解释了为什么之前的 7 帧均匀审计全部未发现
3. **亮度过渡区 t=360-460s 的根本原因部分清晰**：该区包含 (a) 滤芯移除事件、(b) 人类介入、(c) 相机自动曝光调整——三者叠加产生 ±10-14 灰度级波动
4. **滤芯数不是从 4→3 单向变化，而是 4→3→4→3**（加载→卸载→重新加载→再卸载），是一个**可逆的状态序列**——这进一步强化了需要状态机的论点

## 1. 实验目标

构建一个可解释的 temporal/state baseline，用来说明 **Case 2（传送带工位）为什么不能用单帧静态 memory bank，而需要时间状态和动态 memory**。

不运行任何深度学习模型。仅使用帧差、亮度、时间戳 + qwen3-vl-plus 视觉确认。

---

## 2. 方法

### 2.1 状态定义（8 状态）

基于帧差阈值、时序位置和 qwen3-vl-plus 验证，定义 8 个候选状态：

| 状态 | 触发条件 | 物理含义 |
|------|---------|---------|
| **INIT** | t=0s, 视频起始帧 | 系统初始待机 |
| **INDEXING** | frame_diff ≥ 30.87 (mean+3σ) | 传送带分度运动（高帧差尖峰） |
| **TRANSPORT** | frame_diff ∈ [16.0, 30.87) | 滤芯在运输中（中高帧差，运动模糊可见） |
| **DWELL** | frame_diff < 14.0, 非过渡区 | 传送带静止，滤芯停在工位 |
| **PROCESSING** | frame_diff ∈ [14.0, 16.0), 非过渡区 | 检测/加工驻留（气动执行器可能动作） |
| **TRANSITION** | t ∈ [360s, 460s] 亮度变化区（非人类窗口期） | 光照/曝光过渡 + 滤芯数变化 |
| **HUMAN_INTERACTION** | **t ∈ [389s, 397s]（qwen3-vl-plus 确认）** | **人类操作员介入（蓝色洁净服，手持滤芯）** |
| **END_STATE** | t ≥ 435s (滤芯数变化后) | 批次结束/卸载后状态 |

### 2.2 时间平滑

使用滑动中值滤波器（窗口=5帧≈10s）对帧差序列做平滑：

```
diff_smooth[i] = median(diff_raw[i-2 : i+3])
```

目的：
- 抑制短暂尖峰（<1帧的噪声变化）→ 减少虚假状态切换
- 保留持续性变化（真实的分度运动、亮度过渡、人类介入）→ 不丢失真实事件
- 对比 raw state vs smoothed state 的切换频率

### 2.3 事件检测（7 类）

| 事件类型 | 触发条件 |
|---------|---------|
| HUMAN_PRESENT / HUMAN_WINDOW_ENTER | t ∈ [389s, 397s] |
| INDEXING_PEAK | 原始帧差超过 30.87 阈值 |
| STATE_CHANGE | 平滑状态与前一帧不同 |
| BRIGHTNESS_ZONE_ENTER / EXIT | 帧时间戳首次进入/离开 t=360s / t=460s |
| CARTRIDGE_CHANGE_CANDIDATE | t ∈ [316s, 435s] 且亮度低于 115.0 |
| CARTRIDGE_REMOVAL | t ∈ [403s, 405s]（最右位滤芯移除） |
| SPIKE_SUPPRESSED | 原始状态=INDEXING 但平滑后≠INDEXING |

## 5. 为什么单帧检测不够？（五个层次的论证）

### 5.1 同一像素位置在不同时间对应不同物体

Case 4 的透析器位置是固定的。Case 2 相反——滤芯沿导轨移动。**静态 memory bank 的核心假设——"同一位置的像素分布恒定"——在 Case 2 被打破。**

### 5.2 "空"状态的双重性

| 时间 | 滤芯状态 | 是否异常 | 单帧 PatchCore 判断 |
|------|---------|---------|-------------------|
| t=0s | 空位（尚未装载） | **正常** | 高距离 → 误报 |
| t=435s | 空位（可能卸载/丢失） | **未知** | 高距离 → 无法定性 |

### 5.3 发现的人类介入进一步强化论据

t=393s 手持滤芯的帧：单帧检测器看到"导轨上方有模糊的肉色物体 + 第 4 支滤芯悬浮"——这是一个极其异常的外观。但这是**完全正常的补料操作**。只有时序上下文（"2 秒前没有手，2 秒后手消失了"）能判断这是一个短暂的、合法的人类介入，而非系统异常。

### 5.4 正常状态包含多种外观（且可能是可逆的）

Case 2 的滤芯数不是单调的：**4→3→4→3**（卸载→重新加载→再卸载）。单帧检测器看到"3 支滤芯"时无法判断这是正常的批次结束（1 次 4→3）还是异常的中间丢失。

### 5.5 状态机约束是免费的检测信号

本 baseline 仅用帧差+亮度检测到分度运动、状态切换周期、人类介入窗口、滤芯移除事件——这些都是**物理过程的直接信号**，无需任何训练数据或 GPU。

## 6. 为什么 Case 2 需要短期记忆？

**短期记忆定义**：最近 N 帧（~10-60s）的状态和外观信息。本 baseline 的滑动中值平滑（窗口=5帧≈10s）即最简单的短期记忆实现。

| 问题 | 单帧方法 | 带短期记忆的方法 |
|------|---------|---------------|
| 空夹具是正常还是异常？ | 无法区分 | 回看前 5 帧：之前有滤芯→丢失（异常）；一直空→正常 |
| 运动模糊是异常吗？ | 可能产生高分 | 检测到帧差高+状态=TRANSPORT→跳过或降权 |
| 出现人手是异常吗？ | 极度异常 → 最高分 | 检测到 HUMAN_INTERACTION 状态→标记"需人工确认"而非"异常" |
| 滤芯数 3 是正常还是异常？ | 不知道 | 4→3→4→3 的可逆序列→正常操作；4→3 单向→潜在异常 |

**平滑状态从 94 次切换减少到 32 次（-66.0%）**——这是短期记忆最直接的效益。HUMAN_INTERACTION 状态通过时间窗口（而非帧差）定义，因此在中值滤波下保持不变。

---

## 7. 长期记忆应该保存什么？

### 7.1 周期统计

| 内容 | 来源 | 用途 |
|------|------|------|
| 分度运动周期 | 帧差峰值时间间隔 | 检测"运动缺失"或"运动异常频繁" |
| 驻留期持续时间 | DWELL 状态持续时间 | 检测"异常长驻留"（可能卡料） |
| 人类介入频率 | HUMAN_INTERACTION 统计 | 区分"正常补料"(如每批次 1 次)和"异常频繁介入" |
| 滤芯数分布 | 目标检测或人工标注 | 检测"数量异常" |
| 4→3→4→3 模式是否周期性 | 多批次统计 | 判断"重新加载"是标准操作还是单次事件 |

### 7.2 长期记忆 vs 静态 memory bank 的本质区别

| | 静态 Memory Bank (Case 4 式) | 长期时序 Memory (Case 2 需要) |
|------|------|------|
| **存储内容** | 特征向量集合（coreset） | 时序统计 + 状态转换规则 + 人类介入模式 |
| **更新方式** | 手动重建 | 滑动窗口自动更新 |
| **对漂移的响应** | 全部失效，需重建 bank | 检测漂移，自适应或报警 |
| **"正常"的定义** | 一个固定外观分布 | 一组在特定时间合法的状态（含人类介入） |
| **误报来源** | 所有合法但不常见的外观（含人手） | 状态序列违反时序约束 |
| **人类介入处理** | 视为严重异常 | 检测到→标记→请求人工确认→纳入长期记忆 |

## 9. 静态 Memory Bank 在这里会怎样失效？

假设运行与 Case 4 v1 完全相同的 PatchCore+ResNet18 pipeline（预测推演）：

### 9.1 人手 → 极度异常但合法

t=393s 的手持滤芯帧——在工业自动化场景中，人手是一个从未在 Bank Normal 中出现的外观。PatchCore 会给此帧分配极高 anomaly score（可能是全局最高分）。但这是**完全正常的补料操作**。这就是"人类介入=异常"的经典误报模式。

### 9.2 运动模糊 → 系统性误报

Bank Normal 中包含的传送帧有限。Test 流中大量 TRANSPORT 帧的模糊外观与 bank 中大多数清晰帧不同 → 持续高 anomaly score。

### 9.3 空夹具 → 无法定性的高分

t=435s 的 3 滤芯画面 vs bank 中的 4 滤芯画面 → anomaly score 应该上升。但无法判断"正常卸载"还是"异常丢失"。

### 9.4 结论

**PatchCore 在 Case 2 上不是"信号低于噪声"的静默失败（Case 4 模式），而是"信号淹没在合法变化中"的嘈杂失败——包括人手、运动模糊、滤芯数变化、亮度过渡都会产生高分，但绝大部分是合法的。**
