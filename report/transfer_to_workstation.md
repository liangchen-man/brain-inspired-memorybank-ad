# 从 MVTec AD 到工位异常监控的迁移方案

第二周任务不再只是复现 MVTec AD 指标，而是要把 memory bank 范式迁移到真实工业视觉场景。本文件给出第一周实验对第二周的直接指导。

## 任务性质判断

工位异常监控通常不是纯纹理异常检测，而更接近：

- 固定场景下的物体状态异常；
- 工具/工件位置异常；
- 缺失、遮挡、摆放错误；
- 动作流程异常；
- 短时间状态变化异常。

因此它介于物体异常检测、场景状态异常检测和视频异常检测之间。

## 需要回答的八个问题

| 问题 | 第一周给出的启发 | 第二周建议 |
|---|---|---|
| 更像纹理异常还是物体异常？ | `carpet` 是纹理，`bottle` 是物体；工位更接近物体/状态异常 | 不要只用纹理逻辑解释，应关注工具、工件、区域状态 |
| 图像是否具有固定位置结构？ | PaDiM 对固定位置结构友好 | 若固定机位，PaDiM 可作为位置统计基线 |
| 同一位置正常模式是否稳定？ | PaDiM 依赖位置稳定 | 若同一位置变化大，优先 PatchCore/AnomalyDINO |
| 视频单帧是否足够？ | MVTec 是静态图像 | 单帧只能发现空间异常，不能判断流程异常 |
| 是否需要时间状态？ | 第一周没有时间建模 | 第二周至少做帧级分数曲线或滑动平均 |
| 是否需要短期/长期记忆？ | memory bank 是长期正常记忆 | 工位视频应加入短期状态平滑和长期正常库 |
| 是否需要人工反馈？ | MVTec 标注固定 | 真实现场需要人工确认误报并更新记忆库 |
| 静态 memory bank 会不会失效？ | MVTec 分布稳定 | 真实现场会因光照、批次、相机漂移逐渐失效 |

## 建议的第二周实验路线

1. 从老师给的数据中选取正常样本，整理为 MVTec-like 结构。
2. 先跑 PatchCore-teaching，作为轻量 baseline。
3. 再跑 AnomalyDINO-teaching，测试 DINOv2 强特征迁移能力。
4. 如果是视频，输出每帧 anomaly score 曲线。
5. 对帧级分数做 5 帧滑动平均，比较单帧检测和时间平滑。
6. 选 3 个成功样本和 3 个失败样本做定性分析。

## 迁移流程图

```text
MVTec 正常样本
   ↓
建立正常性 memory bank
   ↓
测试异常样本
   ↓
输出 anomaly score + heatmap
   ↓
迁移到工位监控
   ↓
正常工位帧 → 工位 memory bank → 实时帧检测
   ↓
时间平滑 / 短期记忆 / 人工反馈
   ↓
动态更新长期正常记忆
```

## 面向真实工业应用的改进方案

- 建立长期 memory bank：保存稳定正常状态。
- 建立短期 memory：平滑最近若干帧，减少单帧误报。
- 引入人工反馈：人工确认误报后，将样本加入候选正常库。
- 定期更新：按班次、光照、设备状态维护不同正常库。
- 异常分级：把 anomaly score 分为提示、警告、报警三级，而不是单一阈值。
- 区域约束：只检测关键工位区域，减少背景扰动。

## 报告表达建议

第二周报告可以写成：

> 第一周的 MVTec AD 实验证明 memory bank 范式能够在稳定数据集上建立正常性记忆；第二周的工位异常监控进一步检验这种静态记忆能否迁移到真实现场。真实现场与 MVTec 的核心差异在于时序变化、场景漂移和人工反馈需求，因此需要从静态 memory bank 走向动态正常性记忆。
