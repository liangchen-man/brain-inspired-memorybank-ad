## 1. 实验背景

Case 4 为透析器装配工位固定相机监控视频（1920×1080, ~30fps, 1091s），已知 4 个异常时间戳存在侧帽缺失（t=272s T1遮挡，t=949s T1+T5，t=1047s T2+T5，t=1072s T5）。目标：验证 memory bank 范式（PatchCore）从 MVTec AD 迁移到真实工位视频时的检测能力。

共运行三组独立实验，方法复杂度递进：简单像素基线 → 深度特征 PatchCore。

## 3. 结论：当前采集条件下不可稳定检测

**三个独立实验、三种不同方法论、从像素到预训练特征——全部收敛于同一结论：Case 4 视频中侧帽缺失异常在当前采集条件下无法被稳定检测。**

重要限定：

- **此结论仅针对当前 Case 4 视频的采集条件**（相机距离 ~1.5m, 1920×1080, 荧光灯照明, 固定相机但存在微振）。不排除在其他采集条件下（更近相机、更高分辨率、受控光照）检测成功。
- **此结论不否定 PatchCore、DINOv2 或 memory bank 范式本身**。MVTec AD 上 AUROC 0.98+ 的成绩是真实的——只是不能直接迁移到采集条件不足的真实工位场景。
- 这是 **"迁移差距"（transfer gap）的实证**，而非"方法无效"。

## 4. 失败原因分析

### 4.1 缺陷像素占比过低

侧帽实际尺寸约 8–10mm 直径，在 1920×1080 帧中仅占 ~10–15px。在全帧中占比约 **0.004%**，在宽 ROI (710×788) 中占比约 **0.03%**，即使在紧侧帽 ROI (710×200) 中也仅占 **0.1%**。

v2 的 37×37 特征网格上，一个侧帽仅覆盖 **1–2 个特征 patch**——信息量不足以与正常 patch 的统计波动区分。v1 的 28×28 网格更粗。

v3 直接将 40×50px 窗口对准侧帽位置——此时侧帽占窗口的 ~5%——依然无法检出，说明即使在像素级对齐的条件下，信号也被噪声淹没。

### 4.2 特征网格空间分辨率不足

| 方法 | 网格尺寸 | 每格物理覆盖 |
|------|---------|-------------|
| v1 ResNet18 @ 224 | 28×28 | ~25px |
| v2 DINOv2 @ 518 | 37×37 | ~14px |
| 侧帽尺寸 | — | **~10–15px** |

DINOv2 将分辨率提升了近 2 倍，score 动态范围从 [0.788,0.986] 扩展到 [0.513,1.019]，但不足以弥补侧帽仅 1 格的问题。更强的 backbone 改善了表示质量，未改善空间粒度——这是 memory bank 范式在极小缺陷场景的结构性限制。

### 4.3 玻璃面板反射

透析器装配工位前方有透明亚克力防护罩。qwen3-vl-plus 在多个帧中确认存在镜像反射（"Reflection on the left side shows mirrored images"），将左侧透析器的图像投射到不该存在的位置，产生伪影。这些反射随相机角度和照明条件变化，memory bank 将其视为"正常"的一部分，进一步压缩了真实异常信号与正常波动之间的差距。

### 4.4 光照闪烁

工位使用荧光灯管，存在 100Hz 频闪（交流电 50Hz × 2）。在 30fps 采样率下，相邻帧的亮度可能相差 1–3 个灰度级。v3 结果中正常帧间 L2 sigma 达 5–26，其中 T5（边缘位置，受反光影响最大）的 sigma=26.27 是最高值。这种帧间亮度波动是小缺陷信号的**主要噪声源**。

### 4.5 相机微振

工业现场存在机械振动（泵、传送带、装配臂动作），相机即使固定也会产生 1–3px 的亚像素位移。v3 中正常基线 t=180s 自身的 L2=16.7（T5），说明即使"正常 vs 正常"也不是零差异——帧对齐并不完美。侧帽缺失引入的变化（~10px 白→暗）与振动引入的变化（整帧平移 1–3px）在 L2 空间中量级相当。

### 4.6 T1 机械遮挡

t=272s 的 T1 缺帽被机械臂/亚克力罩支架完全遮挡（qwen3-vl-plus 确认 "occluded by mechanical arm"）。这是固定单相机视角的固有盲区——无论算法多强，无法检测不可见区域。工位设计应考虑**多相机覆盖**或**相机视角避开已知遮挡物**。

### 4.7 v2 紧 ROI 的上下文丢失

v2 将 ROI 收窄到侧帽水平带 (y=380-580) 后，t=949s 排名从 #45 **恶化**到 #210。qwen3-vl-plus 事后验证确认：侧帽实际位置在 y≈520-640，v2 的 ROI 下边界 580 **切在了侧帽中间**——部分侧帽（T4/T5）被排除在外。同时，v1 的宽 ROI 包含了透析器之间的空间关系（如 T1 和 T5 同时缺帽的上下文），这些全局信息在紧 ROI 中丢失。过度的 ROI 裁剪可能排除关键上下文。

## 5. 工业改进建议

以下建议按实施成本从低到高排列，均基于本实验的实证发现：

### 5.1 采集质量门控（低成本）

在推理前增加采集质量检查步骤：
- 检测帧间亮度突变（>3 灰度级/帧）→ 丢弃或标记为"不可靠"
- 检测亚像素位移（ORB/SIFT 特征点匹配）→ 超过 1px 偏移则做图像配准
- 仅对通过质量门控的帧做异常检测

**预期效果**：降低正常帧的噪声基线，提升异常 z-score。

### 5.2 光源稳定化（中低成本）

替换荧光灯管为 DC 驱动的 LED 面光源，消除 100Hz 频闪。这是 v3 数据中最大的噪声源（T5 sigma=26.27 主要来自亮度波动）。

**预期效果**：正常帧间 L2 sigma 预计下降 50%–70%，异常 z-score 可能从 1.1 提升到 2–3 区间。

### 5.3 相机靠近 + 提高分辨率（中成本）

当前相机距离约 1–1.5m，侧帽仅占 10–15px。将相机移至 30–50cm 或更换为 4K 摄像头，侧帽可占 40–60px——达到 v2 特征网格的 3–4 格，足以被 DINOv2 patch token 捕获。

**预期效果**：特征网格覆盖从 1 格提升到 3–4 格，PatchCore memory bank 应能可靠区分白帽 vs 暗孔。

### 5.4 侧帽专用视角（中成本）

当前相机为正面广角视角，侧帽在右侧边缘、透视压缩严重。增加一个 45° 侧面相机直接拍摄侧帽区域（y=520-640 水平带），可消除玻璃反射、机械遮挡，并将侧帽投影面积放大 2–3 倍。

**预期效果**：消除 T1 遮挡盲区，降低反射伪影，侧帽信号强度提升 200%+。

### 5.5 机械定位 + 图像配准（中高成本）

在工位增加机械定位销/限位块，确保透析器每次停留位置一致。配合 SIFT/ORB 特征点图像配准，将帧间位移控制在 0.5px 以内。

**预期效果**：消除相机微振引入的整帧 L2 噪声，正常帧 L2 可能降至个位数。

### 5.6 多帧时序降噪（零硬件成本，仅软件）

当前实验已使用 ±0.1s（3 帧）的时序聚合。可进一步扩大窗口到 ±0.5–1.0s（15–30 帧），利用侧帽缺失是**持续性状态**（持续数十秒）而光照/振动是**高频噪声**的特性，通过中值滤波或指数移动平均抑制噪声。

**预期效果**：正常帧 sigma 预计下降 30%–50%，异常 z-score 可能突破阈值。

## 7. 对论文章节的建议

本实验最适合放入论文的 **"Real-World Deployment Analysis"** 或 **"Transfer Gap"** 章节，叙事线：

1. MVTec AD 上 memory bank 方法取得 AUROC 0.98+（第一周）
2. 直接迁移到真实工位视频（Case 4）——三种方法全部失效
3. 不是算法失败，是采集条件限制——用 visibility baseline 证明信号低于噪声地板
4. 列出可操作的工业改进方向（采光、相机、配准、时序）
5. 核心信息：**学术基准 → 工业部署的差距不在模型架构，在采集工程**
