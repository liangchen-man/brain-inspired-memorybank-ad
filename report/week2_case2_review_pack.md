## 2. 六问核心回答

### Q1: 固定机位？

**是。** 相机固定安装，视角不变。

证据：
- qwen3-vl-plus 6 帧审计一致确认："Fixed, eye-level camera, rigidly mounted"
- 帧差分析：背景（聚碳酸酯防护板、塑料箱、铝型材框架）在所有 240 帧中无平移
- 无云台/变焦/画面切换

### Q2: 工件是否移动？

**是。工件沿线性传送导轨做间歇分度运动。**

关键时间点：

| 时间 | 事件 | 证据 |
|------|------|------|
| t=0s | 滤芯静止，传送带空载或初始状态 | qwen3: "No visible motion, all components static" |
| **t=25.7s** | **传送带分度运动**（最大帧差 38.8） | 帧差峰值，qwen3: "conveyor just completed rapid indexing" |
| t=79s | 滤芯运动模糊（主动运输中） | qwen3: "clear horizontal smearing, motion blur confirmed" |
| t=158s | 驻留状态，执行器缩回 | qwen3: "Conveyor appears stationary, uniform spacing" |
| t=197.6s | 检测/驻留位 | qwen3: "quality verification checkpoint, system paused" |
| t=237-316s | 持续驻留 | 帧差低值区 |
| t=363-460s | 亮度变化区（光源/曝光调整？） + 滤芯数减少 | 见 §3 |

**运动模式：间歇分度（indexing shuttle），非连续传送带。** 滤芯在白色夹具中固定，导轨步进移动（约 80-100mm/步），在检测站（气动执行器下方）驻留。

### Q3: ROI 是否稳定？

**不。传统静态 ROI 不适用于 Case 2。**

原因：
- 滤芯沿导轨移动，同一像素位置在不同时间对应不同滤芯或空位
- 如果按 Case 4 方式裁剪固定 ROI，需使用**导轨区域**（rail-level ROI）而非工件级 ROI
- 可考虑 **"追踪 ROI"**：先检测滤芯位置，再在滤芯周围提取局部 ROI——但引入了目标检测前置环节

**候选 ROI 方案**（需 Codex/用户确认）：

| 方案 | 区域 | 优点 | 缺点 |
|------|------|------|------|
| **ROI-A: 导轨全段** | 整条线性导轨 + 夹具 | 覆盖所有滤芯位置，实现简单 | 包含空位和运动模糊帧 |
| **ROI-B: 检测站驻留区** | 气动执行器正下方夹具位 | 滤芯在此外停留最久，状态最稳定 | 只能监控一个位置 |
| **ROI-C: 多工位分块** | 4 个滤芯夹具位置分别 ROI | 逐滤芯独立检测 | 需要标定每个工位坐标 |

### Q4: 单帧检测是否足够？

**不够。** Case 2 至少需要区分以下状态，单帧无法区分：

| 状态 | 单帧特征 | 误判风险 |
|------|---------|---------|
| 空夹具（正常：滤芯尚未到达） | 无滤芯 | → 可能误报为"滤芯缺失" |
| 运动模糊（正常：传送中） | 模糊的低质量图像 | → 可能产生高 anomaly score |
| 滤芯驻留（正常：检测站） | 清晰滤芯 | → 正常 |
| 滤芯缺失（异常：本应在此） | 无滤芯 | → 需时序上下文区分"未到达"vs"丢失" |

**单帧 memory bank 在 Case 2 上的核心矛盾**：正常状态本身包含"有滤芯"和"无滤芯（空位）"两种外观，静态 memory bank 无法区分"正常的空"和"异常的空"。

### Q5: 是否需要时间状态？

**是。Case 2 天然适合时序/状态异常检测。**

以下时间状态转换在本视频中被观察到：

```
[空位/初始] → [滤芯到达] → [驻留/检测] → [分度移动] → [下一工位] → ... → [卸载(?)]
     ↑              ↑            ↑              ↑
   t=0s        t=25-79s     t=158-355s     t=25.7s 峰值
```

特别是 **t=363-460s 亮度变化区 + 滤芯数从 4→3** 表明存在一个明确的"卸载"或"批次切换"事件（候选，需确认）。

**推荐的时序检测方向**：
- **状态机模型**：定义工位的预期状态序列，检测违反状态转换规则的事件
- **时序 memory bank**：按"工位时间窗"分组建立多个正常模板（如"空位期"、"驻留期"、"传送期"各自一个 bank）
- **差分检测**：帧差分析已证明可以检测传送运动（t=25.7s peak），同样可检测异常静止或异常运动

### Q6: 是否适合做短期/长期 memory 讨论？

**是——Case 2 是目前两个 Case 中最适合讨论"动态/时序 memory"的案例。**

适合点：
- **短期 memory**：相邻帧之间的状态一致性（"上一帧有滤芯，这一帧应该还有"）
- **长期 memory**：批次级别的统计特征（"每批次 4 个滤芯，周期约 X 秒，分度间距 Y mm"）
- **memory 漂移**：光照变化（t=363-460s 亮度区）可以触发 memory bank 过期——这是论文的理想讨论素材
- **人类反馈集成**：如果发现异常（如滤芯数从 4→3），系统应请求人工确认（"这是正常的卸载还是异常丢失？"）

**Case 2 vs Case 4 在论文中的角色分配**：

| | Case 4 | Case 2 |
|------|--------|--------|
| **核心信息** | 静态 memory bank 的迁移差距 | 动态/时序 memory 的必要性 |
| **异常类型** | 微小静态缺陷（侧帽缺失） | 状态/序列异常（候选） |
| **实验类型** | 三方法收敛失效分析 | 状态建模 + 时序检测 |
| **关键教训** | 采集方案 > 算法 | 单帧不足以感知状态 |

## 3. 关键事件时间线

### 3.1 帧差峰值事件

| 时间 | 帧差 | 解释 |
|------|------|------|
| **t=25.7s** | **38.83** | **传送带分度运动**（最大峰值，>mean+3σ） |
| t=27.7s | 32.22 | 分度运动余波 |
| t=35.6s | 28.02 | 二次分度或夹具定位 |
| t=393.3s | 26.92 | 亮度变化区内运动 |
| t=381.5s | 24.39 | 亮度变化区内运动 |

阈值 = mean + 3σ = 30.87。仅 t=25.7s 和 t=27.7s 超过。

### 3.2 亮度变化区 (t=360-460s)

连续约 100 秒的亮度波动（±10-14 灰度级），可能原因（候选）：
1. 环境光源自动调节（如 LED 面板调光）
2. 相机自动曝光响应（场景中物体数量/反射率变化触发）
3. 外部遮挡（如操作员经过——但 qwen3 未发现任何人，排除此可能）
4. 工位内部光源切换（如检测站照明灯开启）

此区间恰好也是滤芯数从 4→3 的区间——亮度变化可能与卸载操作相关的设备动作有关。

### 3.3 滤芯数变化（候选，需验证）

- t=0-355s：4 个透明滤芯（无液体，仅吸附剂颗粒可见）
- t=363-395s：4 个滤芯（吸附剂琥珀色更明显——可能是光照角度变化，而非液体注入）
- t=435-472s：**3 个滤芯**（少了一个——可能是卸载完成）

**注**：早期假设（t=435s qwen3 报告"液体注入"）被 t=363s 和 t=454s 的后续审计修正——qwen3 确认吸附剂颜色来自固体颗粒（"solid adsorbent disc/membrane, not bulk liquid"），非液体。t=435s 的"液体"判断是单帧误判。

### 3.4 人员介入

**未发现任何人手/身体介入。** 7 帧深度视觉审计（含边缘/反射/防护板后区域检查）一致返回 "No humans visible"。241 帧全帧差序列中也未出现肤色像素的突然出现特征。

**结论**：Case 2 是全自动化工位，至少在 472s 视频窗口内无人类介入事件——除非人工操作发生在相机视野外（如从侧面/背面装卸滤芯）。

## 4. 异常检测可行性评估

### 4.1 适合的异常类型（候选，需按实际缺陷定义确认）

| 异常类型 | 检测难度 | 推荐方法 |
|---------|---------|---------|
| 滤芯完全缺失（本应在位） | 中 | 时序状态机 + 目标检测 |
| 吸附剂填充异常（量不足/颜色不对） | 高 | ROI-based PatchCore，需要更近视角 |
| 端盖未密封/歪斜 | 高 | 与 Case 4 类似——像素占比小 |
| 条形码缺失/错误 | 中 | 专用 OCR/条码读取模块 |
| 分度运动异常（卡住/跳步） | 低 | 帧差监控 + 周期分析 |
| 夹具未夹紧 | 高 | 需要更高分辨率视角 |

### 4.2 不建议的检测方法

- **全局 static memory bank + PatchCore**：Case 4 已证明在微小缺陷上失效，且 Case 2 的工件移动使静态 ROI 不可行
- **单帧独立评分**：无法区分"空位（正常）"和"丢失（异常）"

### 4.3 推荐的实验方向（候选）

1. **帧差运动检测**（最低成本）：检测分度运动是否在预期时间窗内发生。异常 = 运动缺失（卡住）或运动异常（跳步）
2. **多帧中值背景建模**：从驻留期帧建立正常背景，检测工件区域的结构性变化
3. **ResNet18 特征 + 时序窗口**：对导轨区域使用 Case 4 v1 同款 backbone，但将评分从"单帧 NN 距离"改为"滑动窗口中 NN 距离的变化量"——状态转换处的突变可能携带异常信号
4. **简单状态机**：
   - 定义状态：EMPTY / INDEXING / DWELL / INSPECTION
   - 从帧差和亮度特征自动标注状态
   - 异常 = 状态序列违反预期转换规则

## 5. 与 Case 4 的互补性

论文叙事线（建议）：

```
第一周：MVTec AD 基准 → memory bank 概念验证 (AUROC 0.98+)
    ↓
第二周 Case 4：直接迁移 → 三方法失效 → "采集方案比算法更重要"
    ↓
第二周 Case 2：换个角度 → 不是采集问题，是"单帧不够" → 时序 memory 的必要性
    ↓
讨论：真正的工业部署需要 (1) 采集质量 (Case 4 教训) + (2) 时序感知 (Case 2 教训)
```

Case 4 和 Case 2 共同构成了从"学术 memory bank"到"工业部署"的完整论证链——不是两个独立案例，而是两个互补维度。
