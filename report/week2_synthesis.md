## 1. 总体判断

第二周实验已经形成完整的双案例论证链：

| 案例 | 场景 | 核心问题 | 实验结论 |
|---|---|---|---|
| Case 4 | 固定透析器装配工位，侧帽缺失 | 微小静态缺陷是否能由 memory bank 迁移检测 | 在当前采集条件下不可稳定检测，主要瓶颈是采集质量而非模型大小 |
| Case 2 | 滤芯传送带工位，间歇分度 + 人类介入 | 单帧静态 memory bank 是否足够解释工位状态 | 不足，需要时间状态、短期记忆、长期记忆和人工反馈 |

因此，第二周不应写成“把 PatchCore 用到视频上”，而应写成：

> MVTec AD 证明静态正常性记忆在受控图像基准上有效；真实工位进一步表明，工业部署还需要可观测的采集条件、时序状态理解、动态记忆和人工反馈门控。

## 2. Case 4：采集方案比模型更重要

Case 4 对应固定透析器工位，已知异常是侧帽缺失。实验依次验证了三类方法：

| 方法 | 输入/ROI | 结果 | 解释 |
|---|---|---|---|
| v1 PatchCore + ResNet18 | 宽 ROI，覆盖透析器全貌 | Top-5 0/7，Top-30 1/7 | 粗特征 + 大 ROI 对微小侧帽缺失不敏感 |
| v2 PatchCore + DINOv2 | 紧侧帽 ROI | Top-5 0/7，Top-30 1/7 | 强预训练特征扩大分数范围，但仍无法可靠排序异常 |
| v3 Visibility Baseline | 每个侧帽 40×50px 小 ROI，L2/SSIM/Edge/Hist | 最高 z≈1.1，低于 z=3 阈值 | 异常信号被光照、反射、微振和正常帧间波动淹没 |

关键结论：

- 这不是“PatchCore 无效”，而是当前相机距离、分辨率、玻璃反射、光照闪烁和相机微振使侧帽缺失信号低于噪声地板。
- 在这种条件下，继续换更大模型或租更强显卡不是第一优先级。
- 工业改进应优先考虑：更近相机、专用侧帽视角、稳定 LED 光源、机械定位、图像配准和采集质量门控。

## 3. Case 2：单帧不够，需要时间状态

Case 2 对应滤芯传送带工位。第一版审计发现工件沿导轨间歇分度，静态 ROI 不稳定；v2 时序基线进一步发现短暂人类介入和 `4→3→4→3` 可逆状态序列。

v2 时序基线结果：

| 项目 | 结果 |
|---|---|
| 状态数 | 8 个：INIT、INDEXING、TRANSPORT、DWELL、PROCESSING、TRANSITION、HUMAN_INTERACTION、END_STATE |
| 事件帧 | 44 个 |
| 人类介入 | t=391-395s，蓝色洁净服操作员手持滤芯进入画面 |
| 滤芯数变化 | `4→3→4→3`，不是简单单向 `4→3` |
| 时间平滑 | 状态切换从 94 次降到 32 次，减少 66.0% |

关键结论：

- 同一像素位置在不同时间可能对应不同滤芯、空位、运动模糊或人手，静态空间 memory bank 的“同一位置正常外观稳定”假设不成立。
- “空位”具有双重语义：可能是正常未到达，也可能是异常缺失；必须依赖前后状态判断。
- 人手在单帧上会被 PatchCore 视为强异常，但在工位流程中可能是正常补料或维护动作。
- Case 2 的正常性不是一个固定图像分布，而是一组按时间合法转换的状态。

## 4. 与第一周 MVTec AD 的关系

第一周结果说明：

- PaDiM、PatchCore、SPADE 和 AnomalyDINO 都能在整理好的 MVTec AD 图像上建立正常性记忆并输出异常热力图。
- PatchCore coreset 消融说明 memory bank 可以压缩而不明显损失像素定位能力。
- AnomalyDINO 预训练消融说明强视觉特征空间会显著影响 normality memory 的质量。

第二周进一步说明：

- 强特征和 memory bank 只在输入信号可观测、ROI 合理、状态语义稳定时有效。
- 工业现场的失败不只来自模型能力，还来自采集质量、遮挡、反射、工件运动、合法人类介入和状态切换。
- 因此，真实工业异常检测系统不能只保存静态正常 patch，还要保存时间状态和更新规则。

## 5. 类脑计算解释

可以将本项目的最终思想组织为三层正常性记忆：

| 层级 | 类脑含义 | 项目对应 |
|---|---|---|
| 长期记忆 | 稳定的正常经验 | MVTec AD 训练正常样本、PatchCore/PaDiM/SPADE/AnomalyDINO memory bank |
| 短期记忆 | 当前上下文与最近状态 | Case 2 滑动窗口、状态平滑、前后帧关系 |
| 反馈门控 | 防止错误经验进入记忆 | 人类确认 `4→3→4→3` 是否标准流程，确认人手介入是否正常 |

类脑启发不是“记住所有异常”，而是：

1. 先建立正常性记忆；
2. 在当前上下文中判断输入是否可由正常经验解释；
3. 当现场变化时，不立即把新样本并入长期记忆；
4. 通过短期记忆、人工反馈和防污染门控决定是否更新。

## 6. 最终报告建议结构

建议最终报告的第二周部分按以下顺序写：

1. 真实工位任务定义：不是单纯纹理异常，也不是单纯物体异常，而是工位状态异常监控。
2. 视频数据审计：Case 4 与 Case 2 的相机、分辨率、帧率、工位结构。
3. Case 4 迁移实验：PatchCore/DINOv2/像素基线三方法收敛失败。
4. Case 4 失败分析：采集质量、微小缺陷、反射、光照、微振。
5. Case 2 时序实验：8 状态规则基线、人类介入、`4→3→4→3` 可逆序列。
6. Case 2 失败预测：静态 PatchCore 会将合法状态变化误报为异常。
7. 类脑扩展方案：长期 memory bank + 短期状态记忆 + 人工反馈 + 防污染更新。
8. 工业部署建议：采集质量门控、ROI 设计、时序状态机、版本化 memory bank、人工确认机制。
