## 1. 实验题目

- 类脑计算对计算机视觉的启发：从工业异常检测中的 memory bank 范式理解正常性记忆
- 副标题：从 MVTec AD 静态记忆到真实工位动态时序记忆

## 2. 实验背景

- 工业异常检测的核心挑战：缺陷样本稀少、缺陷类型开放、标注成本高
- 自然思路：先学习"什么是正常"，再检测"什么偏离正常"——这正是 memory bank 范式的核心
- 类脑计算视角：人脑在识别异常之前首先建立了正常经验；海马体-新皮层的记忆巩固机制与 memory bank 更新有类比关系
- MVTec AD 作为受控基准：15 类工业纹理/物体，标准训练/测试分割，ground truth mask
- 真实工位视频作为迁移目标：固定相机、真实产品、未标注缺陷、采集条件不受控

## 3. 第一周：MVTec AD 静态 memory bank 实验

### 3.1 实验设计
- 选取 carpet (纹理) 和 bottle (物体) 两类，覆盖纹理异常和物体异常
- 实现四种 memory bank 形式的教学版 runner，不使用外部 repo，确保内部一致性和可解释性
- 统一使用 ResNet18 layer2 特征（除 AnomalyDINO 使用 DINOv2 ViT-S/14），控制 backbone 变量

### 3.2 四种 memory bank 形式
- **PaDiM**：统计式记忆——每个空间位置的多元高斯（均值 + 逆协方差），Mahalanobis 距离判定异常
- **SPADE**：样本式记忆——存储所有正常图像的全局+局部特征，k-NN 检索最相似正常图像后比较差异
- **PatchCore**：代表性 patch 记忆——从大量正常 patch 特征中 coreset 采样，最近邻距离判定异常
- **AnomalyDINO**：强特征 patch token 记忆——DINOv2 自监督预训练的 ViT patch token，coreset + NN

### 3.3 实验结果要点
- 四种方法在 carpet 上均达到 pixel AUROC > 0.98——受控条件下静态 memory bank 有效
- ResNet18 layer2 特征 (64-dim) 足以支撑前三类方法的正常性表征
- DINOv2 128-dim token 特征在 AnomalyDINO 上达到最佳 pixel AUROC (0.98767)
- 四类 memory bank 的存储大小差异 100×：SPADE (56 MB) > PaDiM (13 MB) > AnomalyDINO (1 MB) > PatchCore (0.5 MB)

### 3.4 两个消融实验
- **AnomalyDINO 预训练消融**：pretrained=True (image AUROC 0.99559) vs pretrained=False (0.33427)——相同结构，有/无预训练特征使 memory bank 质量和运行时间完全不同；说明正常性记忆的关键是强特征空间，而非 ViT 结构本身
- **PatchCore coreset 消融**：coreset 2000 (pixel AUROC 0.98151) vs 1000 (0.98130)——memory bank 减半而 pixel AUROC 几乎不变，说明可以通过 coreset 控制记忆规模应对更多正常样本

## 4. 第一周结果总结
- 四类算法在受控 MVTec AD 上验证了"建立正常性记忆→检测偏离"的可行性
- 不同 memory bank 形式体现不同权衡：统计式(PaDiM)精确但内存大、样本式(SPADE)可解释但检索慢、代表性 patch(PatchCore)轻量可压缩、强特征 token(AnomalyDINO)泛化好但依赖预训练
- 核心课程目标达成：理解 memory bank 不是一种具体算法，而是一种范式——不同实现方式对应不同的记忆形式和部署权衡
- 但是——这些实验都在理想条件下（固定光照、对齐图像、标注 GT）；真实工位会怎样？

## 5. 第二周：真实工位迁移

### 5.1 迁移目标
- 两台真实工位固定相机监控视频（各约 8-18 分钟，1920×1080, ~30fps）
- 无 ground truth 异常标注、无训练/测试分割、无受控光照——典型的工业现场条件
- 迁移问题：MVTec AD 上的 static memory bank 能否直接用于工位视频异常检测？

### 5.2 迁移策略
- 用 qwen3-vl-plus 视觉审计替代人工标注（无 GT 可用）
- 帧差+亮度时序分析做初步状态识别
- 仅在必要时运行 PatchCore/DINOv2/像素基线——不做全量扫描
- 所有结论标记"候选，需 Codex/用户确认"——不伪造 ground truth

## 6. Case 4：采集可观测性限制

### 6.1 工位场景
- 5 台透析器固定排列，相机正对工位，已知 4 个异常时间戳存在侧帽缺失
- 侧帽为直径约 10mm 的白色塑料盖，在 1920×1080 画面中像素占比 < 0.1%

### 6.2 三方法实验
- **v1 PatchCore+ResNet18 宽 ROI** (覆盖 5 台全貌)：top-5 命中 0/7，top-30 命中 1/7，score 范围窄 [0.788, 0.986]
- **v2 PatchCore+DINOv2 紧侧帽 ROI**：top-5 仍 0/7，top-30 仍 1/7；DINOv2 518×518 输入, 37×37 特征网格仍覆盖不到 ~50px 异常
- **v3 像素可见性基线**：每个透析器右侧帽 40×50px 子 ROI，L1/L2/SSIM/Edge/Hist 五种指标，±3 帧时序聚合，25 帧正常中值参考——**4 个异常时间戳全部不可检测**，最高 z=+1.1（远低于 z=3 阈值）

### 6.3 结论
- **"采集方案比算法更重要"**——这不是 PatchCore 或 DINOv2 的失败，而是采集信号条件不足以支撑任何基于像素或特征的异常检测
- 光照闪烁 (σ=5-26)、相机微振 (1-3px)、玻璃反射淹没 ~10px 的缺帽信号
- 第一优先级改进：更近相机、专用侧帽视角、稳定 LED 光源、机械定位——而非换更大模型或租 GPU

## 7. Case 2：时序状态与动态 memory

### 7.1 工位场景
- 滤芯沿线性导轨做间歇分度运动（传送带 shuttle），4 支透明滤芯含琥珀色吸附剂颗粒
- 相机固定但工件移动——与 Case 4 的静态工位根本不同

### 7.2 8 状态 temporal baseline
- 仅用 frame_diff + brightness + timestamp 做规则驱动状态分类——无深度模型，完全可解释
- 8 状态：INIT / INDEXING / TRANSPORT / DWELL / PROCESSING / TRANSITION / HUMAN_INTERACTION / END_STATE
- 滑动中值平滑（窗口=5帧≈10s）：状态切换从 94 降至 32（-66.0%），INDEXING 尖峰全部被抑制
- 44 个事件帧被检测，覆盖 7 类事件

### 7.3 突破发现：人类介入
- 初步审计（7 帧均匀分布）一致报告"无人类"——采样不足导致的假阴性
- 密集审计（4 帧 qwen3-vl-plus 覆盖 t=381-415s）发现人类在 t=391-395s 介入：蓝色洁净服操作员手持滤芯进入画面
- 人类窗口仅 ~4 秒——2 秒采样间隔下仅 2-3 帧捕获

### 7.4 4→3→4→3 可逆序列
- 滤芯数不是单调的 4→3，而是 4→3→4→3（卸载→人工重新加载→再卸载）
- 可逆性是最强的"需要时序状态机"论据——单帧检测器看到"3 支"时无法判断是正常的批次结束还是异常的中间丢失

### 7.5 单帧为什么不够（五层论证）
1. **像素-物体解耦**：同一像素位置在不同时间对应不同滤芯、空位或人手
2. **双重语义**："空位"可以是正常的（尚未到达）或异常的（丢失）
3. **人手 = 极度异常但合法**：t=393s 手持滤芯在单帧上是极其异常的外观，但在流程中是正常补料
4. **正常 = 一组时序合法状态**：4⇄3 是可逆的正常模式，不是固定外观分布
5. **状态机约束 = 免费检测信号**：非预期的状态转换本身就是异常——不需要训练数据

## 8. 类脑计算讨论

### 8.1 三层记忆框架
| 层级 | 类脑类比 | 项目对应 |
|------|---------|---------|
| **长期记忆** | 新皮层/海马体巩固后的稳定经验 | MVTec AD 训练的静态 memory bank (PaDiM/PatchCore/SPADE/AnomalyDINO) |
| **短期记忆** | 工作记忆 / 前额叶维持的上下文 | Case 2 滑动窗口 + 状态平滑 + 前后帧关系 |
| **反馈门控** | 前额叶-海马体交互，防止错误记忆巩固 | 人工确认 4→3→4→3 是否标准流程，决定是否将新状态纳入长期记忆 |

### 8.2 为什么静态 memory bank 会失效
- **Case 4 模式（静默失效）**：正常记忆库的分数范围窄，异常帧无法被可靠区分——不是记忆库"忘记"了正常，而是采集信号中根本没有区分正常/异常的信息
- **Case 2 模式（嘈杂失效）**：人手、运动模糊、滤芯数变化、亮度过渡——全部产生高分——但绝大多数是合法操作；记忆库无法区分"合法的状态变化"和"真正的异常"

### 8.3 防污染门控
- 不能把所有新样本自动纳入记忆库——否则异常也会被"记住"为正常
- 需要人工反馈决定：这个新状态是"合法扩展的正常"还是"异常"
- 版本化的 memory bank：更新历史可追溯，出问题时可以回滚

### 8.4 类脑启发不是"记住所有异常"
- 先建立正常性记忆
- 在当前上下文中判断输入是否可由正常经验解释
- 当现场变化时，不立即把新样本并入长期记忆
- 通过短期记忆、人工反馈和防污染门控决定是否更新

## 9. 前沿方法对照

- **DMAD**：双记忆库（正常+异常），启发"人工反馈后的异常案例库"概念
- **SimpleNet**：合成异常制造反例，启发"正常记忆库边界需要显式定义"
- **Dinomaly**：Transformer 统一模型，与 AnomalyDINO 属同一技术路线——从逐类 bank 到统一表征
- **AnomalyGPT**：VLM 语义异常解释，对应 Case 2 的 qwen3-vl-plus 手动审计——未来可自动化
- 本项目不实现以上方法，原因：课程范围、算力约束、瓶颈已通过采集和时序维度充分诊断

参考文献：见 `report/frontier_methods_comparison.md`

## 10. 工业部署建议

- **采集质量门控**：在模型之前设置采集质量检测（亮度/对焦/振动/遮挡），采集不合格的帧不做异常推理
- **ROI 设计**：根据缺陷物理尺寸和相机分辨率选择 ROI——MVTec AD 整图级别的 ROI 在真实微小缺陷上不够
- **时序状态机**：对于有运动部件的工位，状态机是最低成本的状态感知方式——帧差+亮度已经可以区分 8 个状态
- **版本化 memory bank**：每次更新记录时间戳、触发条件和人工确认状态——可回滚
- **人工确认机制**：HUMAN_INTERACTION 状态和相关事件应标记为"需确认"而非直接报警

## 11. 局限性

- MVTec AD 仅使用 carpet 和 bottle 两类，未覆盖全部 15 类
- Case 4 无 ground truth 标注——所有异常时间戳为候选，基于 qwen3-vl-plus 视觉确认，非实际工位日志
- Case 2 的 8 状态分类基于 240 帧单一视频——阈值和数据来自单次采集，泛化需要多批次验证
- 人类介入窗口 (t=389-397s) 仅 4 帧，可能错过窗口外的人类操作
- 未运行真实 PatchCore 实验验证 Case 2 的"嘈杂失效"预测——该预测基于逻辑推演，非实验结果
- 所有 Case 2 结论标记为"候选，需 Codex/用户确认"

## 12. 总结

- 第一周在 MVTec AD 上验证了四类静态 memory bank 的可行性和各自权衡
- 第二周通过两个真实工位案例揭示了静态 memory bank 在工业现场的两种失效模式
- Case 4：失效来自采集可观测性——"采集方案比算法更重要"
- Case 2：失效来自缺少时序上下文——"单帧不够，需要时间状态和动态记忆"
- 类脑计算为这个演替提供了理论框架：长期记忆 (memory bank) + 短期记忆 (sliding window) + 反馈门控 (human confirmation)
- 项目的核心贡献不是追 SOTA，而是从第一性原理理解正常性记忆的边界条件——在什么时候、为什么、怎么失效
- 前沿方法（DMAD/Dinomaly/扩散模型/VLM）作为未来工作方向，但本项目揭示的采集和时序两个瓶颈对任何方法的工业落地都适用
